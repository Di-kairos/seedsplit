# seedsplit

Распределить секрет на доли (Shamir Secret Sharing) — часть экосистемы
[Paranoid Tools](https://github.com/Di-kairos/paranoid-tools).

Разбить seed-фразу, пароль или ключ на N долей так, что любые T из них восстанавливают
секрет, а T-1 долей не дают о нём **ничего**. Чтобы один носитель или бэкап не был
единой точкой отказа и компрометации.

> **Статус: ранний (v0.1.0, scaffold).** Готов каркас: вендоринг общего ядра + dispatcher.
> Логика (`split`, `combine`, арифметика GF(256)) — в следующих паках.

## Использование

```bash
seedsplit split   # разбить секрет на N долей, порог T  (pack 2)
seedsplit combine # восстановить секрет из T долей       (pack 2)
seedsplit version
```

## Архитектура

- Single-file Bash, ноль зависимостей. Shamir над GF(256) реализуется на чистом Bash;
  ГСЧ — `/dev/urandom`.
- Общее ядро (`lib/common.sh`) **вендорится** из securetrash inline, пиннуто к git-ref;
  `tools/vendor-common.sh --check` ловит дрейф в CI. См. `paranoid-tools/README.md`.

## Scope & limitations

> Раздел будет дополнен по мере реализации ядра. Принцип экосистемы: честно про пределы —
> здесь особенно легко скатиться в снейкойл:
> - **качество долей = качество ГСЧ** — берём `/dev/urandom`, не самопал;
> - **секрет в `argv` виден в `ps`** — ввод только через stdin/файл, не аргументом;
> - **доли безопасны ровно настолько, насколько безопасно ты их ХРАНИШЬ и РАЗНОСИШЬ;**
>   порог защищает от утечки <T долей, но не от ПОТЕРИ ≥(N-T+1) долей — это твоя
>   ответственность за географию и носители;
> - **совместимости со SLIP-39 / аппаратными кошельками ПОКА НЕТ** — полноценный SLIP-39
>   (1024-словарь + RS1024 + шифрование) это отдельное решение по объёму, не «ноль
>   зависимостей»; в scaffold честно помечено как scope-решение на будущий пак.

## Windows-эквивалент

Планируется во вторую очередь: тот же Shamir над GF(256) на PowerShell, `RNGCryptoServiceProvider`
вместо `/dev/urandom`. Порт — как у securetrash.
